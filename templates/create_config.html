How Configuration Files Work

Configuration files are used to define how the test automation interacts with different platforms. Each configuration file includes:

  • WebDriver Options: Browser window settings and other WebDriver options
  • Platform Information: Name, type, and base URL for the platform
  • Login Details: Whether login is required and corresponding fields
  • Selectors: CSS or XPath selectors for important UI elements (year dropdown, model field, etc.)

After creating a configuration, you'll need to edit it to add specific selectors for the website you're testing.
